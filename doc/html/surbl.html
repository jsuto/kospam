SURBL support

Surbl is a kind of RBL list. You may check every URI in your incoming email against the SURBL database.

Clapf takes the last two part of the domain name, eg. aaaa.yourdomain.com -> yourdomain.com joined with the surbl_domain variable: yourdomain.com.multi.surbl.org.
Dotted IP-addresses will be reversed: http://1.2.3.4/ -> 4.3.2.1.multi.surbl.org.

If a positive match is found clapf adds a special spammy token thus the overall spamicity will be higher and possibly reach the spam limit. See http://www.surbl.org/ for more details about the SURBL system or http://www.uribl.com/index.shtml about uribl.

Of course you may set up your own rbl domain to check URLs against it and query multiply RBL lists with a comma separated list of domains, eg. 

surbl_domain=multi.surbl.org,multi.uribl.com,spamuri.yourdomain.com
Let's say you want to create your own local uribl list. Edit perl/uribl.pl script and set the $dom and @dnsip variables. You need the Net::DNS Perl module to run this script.

After you have created the token database, run the following command: 

grep ^URL\* spam.tmp | perl/uribl.pl > rbl.data
This extracts every URL from the spam folder and check if they are valid domains. If so, create a DNS entry using tinydns format, eg: 
+alwaniti.com.spamuri.yourdomain.com:127.0.0.2:1800
You should create the SOA and NS records of your uribl zone, eg: 
Zspamuri.yourdomain.com:postmaster.yourdomain.com:2006051601:28800:10800:604800:10800::
.spamuri.yourdomain.com:1.2.3.4:ns1.spamuri.yourdomain.com:259200
.spamuri.yourdomain.com:1.2.3.5:ns2.spamuri.yourdomain.com:259200
And you have to append the rbl.data file to the 'header' of your zone, so your dns data will look like this: 
Zspamuri.yourdomain.com:postmaster.yourdomain.com:2006051601:28800:10800:604800:10800::
